13. Hybrid recommendation gate switched from strict to soft policy.
Updated `services/pipeline/scripts/run_recommend_hybrid.py`:
- Added `--wardrobe-floor-score` (default `0.45`) in addition to `--wardrobe-min-score` (`0.55`).
- New behavior:
  - If best wardrobe score `< 0.45`, fallback to Polyvore.
  - If best wardrobe score is in `[0.45, 0.55)`, keep wardrobe presence (soft keep) and fill remaining slots.
  - Scores `>= 0.55` remain strong wardrobe matches.
Result:
- Reduced excessive Polyvore takeover while preserving quality fallback.
- `quality_gate` metadata now reports `wardrobe_floor_score` and `wardrobe_strong_candidates`.
