## 3) End-to-End Pipeline (Current)
1. Input image(s) are loaded.
2. External-upload auto-crop stage (YOLO pose):
   - Detects person/keypoints and infers visibility.
   - Auto-crops upper/lower region by semantic hint (`tops`/`bottoms`) when reliable.
   - Falls back to original image if uncertain or unavailable.
3. Metadata/category resolution:
   - If image is inside dataset `images/` and filename matches item ID: use Polyvore metadata.
   - Else: use category fallback model (DeepFashion B checkpoint), filtered by semantic hint (`tops` / `bottoms`) and confidence gate.
4. Foreground mask is generated using selected background-removal backend.
5. Visual features are extracted on mask region (color + brightness-related metrics).
6. Pattern is predicted from image model (if loaded).
7. Pair model computes base compatibility score from learned embeddings.
8. Final compatibility score is computed by weighted fusion.
9. Label assigned by score band:
   - `Excellent Match` if `score >= excellent_threshold`
   - `Good Match` if `threshold <= score < excellent_threshold`
   - `Borderline Acceptable` if `borderline_threshold <= score < threshold`
   - `Weak Match` if `weak_threshold <= score < borderline_threshold`
   - `Mismatch` otherwise

## 4) Compatibility Calculation (Exact)

### 4.1 Final Fusion
Let:
- `M = model_score`
- `T = type_prior_score`
- `C = color_score`
- `B = brightness_score`
- `P = pattern_score`

All components are clipped to `[0, 1]`, then:

`final = w_m*M + w_t*T + w_c*C + w_b*B + w_p*P`

Current config weights (`configs/pipeline_config.json`):
- `w_m = 0.60`
- `w_t = 0.15`
- `w_c = 0.12`
- `w_b = 0.08`
- `w_p = 0.05`

### 4.2 Model Score (`M`)
- Each image is encoded to embedding `e` (ResNet backbone + projection layer).
- For a pair:
  - `z = concat(e_top, e_bottom, |e_top - e_bottom|, e_top * e_bottom)`
  - `logit = head(z)`
  - `M = sigmoid(logit)`

### 4.3 Type Prior Score (`T`)
- Built from Polyvore disjoint train compatibility data.
- For each top/bottom category pair `(tc, bc)`:
  - `pos = number of positive pairs`
  - `neg = number of negative pairs`
  - Laplace-smoothed prior:
    - `T(tc,bc) = (pos + alpha) / (pos + neg + 2*alpha)`, with `alpha = 1.0`
- Lookup key: `"tc|bc"`.
- Default is `0.5` if unknown/missing.
- Important current rule:
  - type prior is used only when category source is `metadata`
  - fallback image-model categories do not drive type prior IDs

### 4.4 Color Score (`C`)
- Dominant colors are extracted from masked foreground pixels.
- Neutral near-white/near-black low-saturation pixels are filtered.
- Top-1 dominant color from each item is compared:
  - If either color is neutral (`black/white/gray`): base `0.82`
  - Else by hue distance `d`:
    - `d <= 20`: base `0.76`
    - `20 < d <= 55`: base `0.90`
    - `150 <= d <= 210`: base `0.88`
    - otherwise: base `0.64`
- Add `+0.06` if value difference in `[25, 110]`.
- Clamp to `[0,1]`.

### 4.5 Brightness Score (`B`)
- From masked LAB/HSV metrics:
  - brightness, contrast, saturation
- Differences:
  - `bdiff = |brightness_top - brightness_bottom|`
  - `sdiff = |saturation_top - saturation_bottom|`
  - `cdiff = |contrast_top - contrast_bottom|`
- Gaussian components:
  - `bscore = exp(-((bdiff - 22)^2) / (2*20^2))`
  - `sscore = exp(-(sdiff^2) / (2*30^2))`
  - `cscore = exp(-(cdiff^2) / (2*20^2))`
- Final:
  - `B = 0.5*bscore + 0.3*sscore + 0.2*cscore`
- Clamp to `[0,1]`.

### 4.6 Pattern Score (`P`)
Primary path (image model enabled):
- Pattern predictor outputs multi-label probabilities.
- Determine `patterned` by threshold (`0.35` currently).
- Base rule:
  - both solid: `0.84`
  - one patterned, one solid: `0.90`
  - both patterned:
    - same top label: `0.78`
    - same pattern group: `0.72`
    - different groups: `0.63`
- Distribution similarity blend:
  - cosine similarity `sim` over normalized pattern probability vectors
  - `alpha = 0.4` if both patterned else `0.2`
  - `blended = (1-alpha)*base + alpha*(0.55 + 0.45*sim)`
- Confidence adjustment when both patterned:
  - `blended += 0.05*(avg_top_prob - 0.5)`
- Clamp to `[0,1]`.

Fallback path (if pattern model unavailable):
- text-tag heuristic from metadata text:
  - solid-solid: `0.84`
  - one solid with one patterned: `0.90`
  - overlapping pattern tags: `0.78`
  - otherwise: `0.63`

## 5) Retrieval Calculation (Top-K)
1. Encode query image embedding.
2. Build candidate pool by semantic target:
   - `top2bottom` uses `bottoms`
   - `bottom2top` uses `tops`
3. Cosine shortlist stage (fast prefilter).
4. Rerank shortlist using same fusion score as pair mode.
5. Return top-k final ranked candidates.

Current retrieval defaults:
- candidate splits: `train + valid`
- `test` candidates disabled by default (`allow_test_candidates=false`)

## 9) 2026-02-23 Fairness Trial (Applied)

### 9.1 Weight Rebalance
Updated default fusion weights in `configs/pipeline_config.json`:
- model: `0.60` (was `0.65`)
- type_prior: `0.15`
- color: `0.12` (was `0.10`)
- brightness: `0.08` (was `0.05`)
- pattern: `0.05`

### 9.2 Five-Band Labeling
Added 5-level labeling:
- `Excellent Match` if `score >= excellent_threshold`
- `Good Match` if `threshold <= score < excellent_threshold`
- `Borderline Acceptable` if `borderline_threshold <= score < threshold`
- `Weak Match` if `weak_threshold <= score < borderline_threshold`
- `Mismatch` otherwise

Current thresholds:
- `threshold = 0.62`
- `borderline_threshold = 0.55`
- `weak_threshold = 0.45`
- `excellent_threshold = 0.72`

### 9.3 Adaptive Weighting for External/OOD Cases
When category source is not fully metadata (e.g., fallback image model), model dominance is reduced:
- default model penalty: `-0.10`
- stronger penalty for low-confidence fallback: `-0.15`
- removed weight is redistributed to visual terms (`color`, `brightness`, `pattern`)

Reason: avoid over-penalizing visually good pairs where learned model confidence is lower on external images.
