# Fix Log - 2026-03-02

## Scope

This file records the pipeline fixes made on March 2, 2026 for `OutfitCompatibility`, focused on `Mix & Match` full-body handling, clothing isolation, color robustness, and multi-color garment support.

- `services/pipeline/configs/pipeline_config.json`
- `services/pipeline/src/outfit_pipeline/autocrop.py`
- `services/pipeline/src/outfit_pipeline/config.py`

## Fixes Completed

### 1. Full-Body Crop Tightening

- Added `full_body_tighten_ratio` to autocrop config.
- Replaced the initial fixed crop shrink with an adaptive full-body tighten step.
- Tightening now scales based on how large the detected person is in frame.
- Tightening also uses semantic keypoints:
  - `tops` uses upper-body anchors
  - `bottoms` uses lower-body anchors
- Horizontal bounds are now based more on relevant body landmarks instead of only the full person box.
- Vertical trimming is asymmetric:
  - `tops` trims more lower spill
  - `bottoms` trims more upper spill
- Crop cache keys were updated so old cached crops do not mask the new behavior.

### 2. Clothing-Aware Segmentation

- Changed the default foreground method from `u2net` to `segformer`.
- Added semantic-aware clothing masks:
  - `tops` prefer upper-garment labels
  - `bottoms` prefer lower-garment labels
- This reduces exposed skin being treated as clothing during feature extraction.
- `segformer` now degrades to `u2net` or `rembg` if the backend is unavailable.
- Replaced the previous weak-mask full-image fallback with safer behavior:
  - soft mask expansion for small masks
  - localized fallback region instead of a full-frame mask
- Mask cache keys were updated to include semantic context and the new fallback behavior version.

### 3. Lighting-Robust Color Extraction

- Added a masked color-constancy step before dominant-color extraction.
- Dominant colors are now computed from a lightly white-balanced version of the masked garment region.
- White-balance correction is intentionally damped for highly saturated garments so bold colors are not over-corrected.
- Added lighting quality signals to visual features:
  - `lighting_reliability`
  - `white_balance_shift`
- Brightness and contrast still come from the original masked image.
- Color and brightness weights are now downweighted automatically when lighting is unreliable.
- Response details now expose lighting diagnostics for both garments and the pair.

### 4. Dual-Primary and Multi-Color Garments

- Added a filtered meaningful-palette step so tiny accents and near-duplicate shades are less likely to become false secondary colors.
- Harmony scoring now uses meaningful palettes instead of raw top clusters alone.
- The dominant-color anchor in color harmony is reduced when one or both garments are clearly multi-color.
- The backend response now includes:
  - `top_secondary_color`
  - `bottom_secondary_color`
  - `top_color_mode`
  - `bottom_color_mode`
  - `top_color_palette`
  - `bottom_color_palette`
- Existing primary-color fields were retained for compatibility.

## Validation Performed

- No full end-to-end image tests were run as part of these changes.

## Notes

- These changes are backend and pipeline focused.
- Frontend display logic has not yet been updated to surface the new secondary color and palette fields.
